SQL

--- 데이터는 대소문자를 구분한다. 

* 참고 파이썬, Linux는 대소문자 구별함 (즉 대부분의 개발 언어는 대소문자를 구별, 안하는것도 있음 ex) R )



주의 꼭 문장 종결자 ; 빼먹지 않게 , 컬럼 구분자는 , 로 구문한다 



Select(데이터 조회 명령어)
select 구문은 select ~ from 구조로 작성이 됨

select * from 테이블명에서 *는 테이블 내 모든 컬럼을 의미함 


select 컬럼명 from 테이블명; 
하면 원하는 컬럼만 선택하여 조회가 가능하다. 
- 컬럼 조회 순서도 임의로 변경이 가능하다.  내가 원하는 조회 순서대로 컬럼명을 입력하면 됨 
- 동일한 컬럼 여러번 조회가 가능하다. 그 이유는 여러가지 이유가 있다. 
- 테이블내 존재하지는 않지만 임의적으로 만들어서 조회할 수 있다. 
 (컬럼명을 임의로 입력하면 만들어져 조회 가능) 

SQL 구문 내에서 산술연산( + - /  *) 가능하다 
- 단, 문자형 빼고 숫자형 날짜형만 가능함

 




테이블 구조를 조회하는 명령어 
desc 테이블명 하면 테이블 구조를 조회한다 



* 참고 여기서 desc는 (describe) 약자로 사용이되고
desc가 select ~ order by ~ desc 이면 정렬 방향을 의미한다.
정렬 방향을 의미 할 때는 descending의 약자로 사용이 되었다 .
어느 위치에 있는지 잘 확인 할 것 .


















query / 데이터 / select(조회)
DML / 데이터 / insert(입력), update(수정), delete(삭제), merge(insert+update)
TCL / 트랜잭션 / commit(저장), rollback(취소), savepoint(중간 저장점)
DDL / object / create(생성), alter(변경), drop(object삭제), truncate(절삭)
DCL / 권한 / grant(부여), revoke(취소)

위에 내용중 query, DML, TCL 정도는 개발자가 다루고 DDL, DCL은 관리자가 주로 다룬다
